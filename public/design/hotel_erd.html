회원 & 인증 시스템
가입, 로그인, 권한 관리
기반 인프라

🔐 소셜 로그인

카카오, 네이버, 구글 OAuth 2.0 연동. JWT access/refresh token 발급. 자동 회원가입 처리.

👤 회원 유형 관리

게스트(예약자), 호스트(숙소 등록자), 관리자 역할 분리. 호스트 전환 신청 및 승인 프로세스.

✉️ 이메일 인증

가입 시 이메일 인증 메일 발송. 토큰 만료 처리 (24시간). 재발송 기능.

🌍 다국어/다통화

사용자 언어·통화 설정 저장. KRW, USD, JPY 등 환율 적용. i18n 지원.

숙소 & 객실 관리
호스트 숙소 등록 및 관리
핵심 기능

🏨 숙소 등록/수정

숙소명, 타입(호텔/펜션/모텔), 주소, 설명, 체크인/아웃 시간, 편의시설, 이미지 등록.

🛏 객실 유형 관리

룸타입별 가격·베드타입·정원·면적 설정. 뷰 타입, 흡연여부 구분. 룸 수량 관리.

📆 재고/가격 캘린더

날짜별 가용 객실 수 관리. 특정일 가격 오버라이드 (성수기/이벤트). 휴업일 설정.

🖼 이미지 관리

S3 다중 이미지 업로드. 대표 이미지 지정. 순서 변경 (drag & drop). WebP 변환.

✅ 관리자 승인

신규 숙소 등록 시 관리자 검토 후 활성화. 미승인 숙소는 검색 노출 제외.

📊 호스트 대시보드

예약 현황, 수익 통계, 리뷰 관리, 객실 가용률 차트 제공.

🔍 통합 검색

🔥 추천 / 인기

📋 예약 프로세스

날짜선택 → 객실선택 → 정보입력 → 결제 → 확인의 단계별 플로우. 예약번호 자동 생성.

🔒 재고 잠금

💳 PG 연동

토스페이먼츠/KG이니시스/아임포트 연동. 카드/카카오페이/네이버페이 지원. 웹훅 처리.

🎟 쿠폰/할인 적용

쿠폰 코드 입력 및 유효성 검증. 정률/정액 할인. 최대 할인 금액 상한 처리.

❌ 취소 & 환불

정책별 취소 수수료 자동 계산. PG사 환불 API 연동. 환불 처리 이력 관리.

예약 완료 시 이메일/SMS 발송. QR코드 포함 PDF 바우처 생성. 영수증 이메일.

⚡ 성능 최적화 인덱스

  • properties 테이블: (city, is_approved) 복합 인덱스
  • room_availability: (room_type_id, date) 유니크 인덱스
  • bookings: (user_id, status), (check_in_date, check_out_date) 인덱스
  • reviews: (property_id, is_visible) 인덱스
  • 위치 검색용 위도/경도 공간 인덱스 (PostGIS)

🔒 동시성 처리 전략

  • 예약 시 Redis로 객실 재고 임시 잠금 (10분)
  • 결제 완료 시 DB 트랜잭션으로 available_count 감소
  • Optimistic Locking으로 이중 예약 방지
  • Idempotency Key로 결제 중복 요청 방지

📦 캐싱 전략 (Redis)

  • 인기 숙소 목록: TTL 30분 캐싱
  • 숙소 상세 정보: TTL 1시간 (수정 시 무효화)
  • 환율 정보: TTL 1시간
  • 세션 및 JWT 블랙리스트 관리
  • 검색 자동완성 키워드 캐싱

💰 가격 정책 설계

  • room_types.base_price: 기본 단가
  • room_availability.price_override: 날짜별 특별가
  • bookings에 original_price, discount_amount 분리 저장
  • 통화는 항상 원본 통화로 저장 + 환율은 별도 처리
  • 가격 이력 테이블로 분쟁 대비

🛡 보안 고려사항

  • 비밀번호: bcrypt (cost 12이상) 해싱
  • 결제 카드 정보는 PG사에만 저장, DB 비보관
  • 모든 금액 변조 방지: 서버에서 최종 금액 재계산
  • 호스트-숙소 소유권 API 레벨에서 검증
  • 리뷰는 실제 예약 완료건만 작성 가능하도록

📈 확장성 고려

  • 이미지 엔티티 polymorphic 구조로 유연하게 확장
  • 알림은 별도 notifications 테이블 + 큐(SQS/RabbitMQ)
  • 검색은 ElasticSearch 분리 권장 (대용량 시)
  • 호스트 정산은 settlements 테이블 별도 관리
  • 이벤트 로그/감사 테이블로 변경 이력 추적

🗃 추천 기술 스택

  • DB: PostgreSQL (위치 검색 PostGIS, JSON 지원)
  • 캐시: Redis (재고 잠금, 세션, 캐싱)
  • 스토리지: AWS S3 + CloudFront (이미지 CDN)
  • 검색: Elasticsearch (전문 검색 확장 시)
  • 메시지: SQS + Lambda (알림, 이메일 비동기 처리)

📋 소프트 딜리트 전략

  • users, properties, reviews는 deleted_at 컬럼으로 소프트 딜리트
  • bookings는 절대 삭제 금지 (결제/법적 이유)
  • ORM에서 default scope로 deleted_at IS NULL 필터 적용
  • 개인정보는 탈퇴 후 30일 후 하드딜리트 스케줄러
