# clkhotel 배포 담당자용 개발/배포 계획 정리

## 1. 현재 프로젝트 진행 상태 (요약 보고)

`clkhotel`는 **Ruby on Rails 8 기반 호텔 예약 서비스**이며, 기능 개발이 꽤 진행된 상태입니다.

구현된 주요 기능(확인 기준):
- 회원가입 / 로그인 / 로그아웃
- 숙소 목록 / 검색 / 상세
- 예약 생성
- 호스트 예약 관리 (확정 / 거절 / 삭제)
- 결제 단계 / 예약 상태 전이
- 리뷰 작성
- 호스트 숙소 관리 / 객실 유형 관리
- 객실 재고관리 (`room_availabilities`)
- 호수 번호 관리 및 예약 확정 시 호수 지정
- 이미지 업로드 (Active Storage 기반)
- 카카오 알림톡 연동 구조 (mock/API 구조)

배포 준비 흔적:
- `Dockerfile` 존재
- `kamal` gem 사용
- `config/deploy.yml` 존재 (샘플값 상태)

즉, **기능 개발은 상당히 진행되어 있고**, 배포는 **설정 정리 후 진행 가능한 단계**입니다.

## 2. Ruby on Rails가 이 프로젝트에 어떻게 적용되어 있는지 (쉽게 설명)

이 프로젝트는 Rails의 전형적인 **MVC 구조**로 만들어져 있습니다.

### 2-1. Routes (URL 설계)
- 파일: `config/routes.rb`
- 역할: 브라우저 요청 URL을 어떤 기능으로 연결할지 정의

예시:
- `/` → 메인 화면
- `/properties` → 숙소 목록
- `/bookings/:id` → 예약 상세
- `/host/...` → 호스트 관리 기능

### 2-2. Controllers (요청 처리)
- 위치: `app/controllers/`
- 역할: 사용자의 요청을 받아서 DB를 읽고/저장하고, 어떤 화면을 보여줄지 결정

예시:
- `BookingsController` → 예약 생성/취소/완료
- `PaymentsController` → 결제 처리
- `Host::DashboardController` → 호스트 예약 확정/거절/호수 지정

### 2-3. Models (DB + 핵심 비즈니스 로직)
- 위치: `app/models/`
- 역할: DB 테이블과 연결되고, 검증/계산/규칙을 담당

예시:
- `Booking`, `Property`, `RoomType`, `RoomAvailability`, `Payment`, `Review`, `User`

이 프로젝트의 핵심 로직 예:
- 예약 가능 여부 검증
- 날짜별 재고 검사
- 가격 계산
- 예약 상태 전이 (`예약확정 대기 -> 결제/취소 단계 -> 예약 확정`)
- 호수 번호 중복 배정 방지

### 2-4. Views (화면)
- 위치: `app/views/`
- 파일 형식: `.html.erb`
- 역할: Rails 서버가 HTML을 직접 만들어 브라우저에 전달

중요:
- 이 프로젝트는 React/Next 프론트 분리형이 아니라 **Rails 서버 렌더링 모놀리스**입니다.
- 그래서 **Vercel보다 VPS/Docker 배포와 더 잘 맞습니다.**

### 2-5. Rails 추가 기능 사용
- `Active Storage`: 이미지 업로드/관리
- `Turbo / Stimulus`: UI 인터랙션 보강 (Hotwire)
- `Solid Queue / Cache / Cable`: Rails 8 기본 스택 사용

## 3. 현재 배포 관점에서 중요한 기술 상태

### 3-1. DB (현재)
- `SQLite` 사용 중
- 파일: `config/database.yml`
- 운영 DB도 기본값이 `storage/production.sqlite3`

의미:
- **단일 서버 초기 배포에는 유리** (설정 단순)
- 확장성/운영 안정성은 Postgres보다 낮음

### 3-2. 이미지 저장 (현재)
- `Active Storage`가 `:local` 사용
- 파일: `config/storage.yml`, `config/environments/production.rb`

의미:
- 서버 로컬 디스크에 저장됨
- 서버 교체/디스크 문제 시 유실 가능
- 초기 배포는 가능, 장기 운영은 S3/R2 권장

### 3-3. 배포 방식 (현재 준비 상태)
- `Kamal + Docker` 기반 배포 준비됨
- 파일: `config/deploy.yml`, `Dockerfile`

의미:
- **이 프로젝트는 VPS에 올리는 방식과 매우 잘 맞음**
- 코드 수정 없이도 배포 전략 수립이 쉬움

## 4. 내 목표 기준 점검 (중요)

목표:
1. 코드 수정 최소화
2. 서버를 무료로 올리기 (도메인은 구매)
3. 최종검토

### 결론 요약
- 이 목표 조합에 가장 잘 맞는 방식은:
  - **무료 VPS(Oracle Cloud Always Free) + Kamal + Docker**
- 초기에는 코드 수정 최소화를 위해:
  - **SQLite 유지**
  - **Active Storage local 유지**
  로 시작하는 것이 현실적임

## 5. 추천 배포 전략 (최소 수정 / 무료 우선)

## 추천안: Oracle Cloud Always Free VM + Kamal

### 왜 이게 맞는가
- Rails 모놀리스 구조와 궁합이 좋음
- Docker/Kamal 설정이 이미 프로젝트에 있음
- 무료 VM 사용 가능 (조건/제약 있음)
- 도메인 연결 가능
- 코드 수정 최소화 가능

### 주의사항
- Oracle Free Tier는 계정 생성/리소스 확보가 까다로울 수 있음
- 무료 리소스 품질/가용성은 유료보다 낮을 수 있음
- 운영 백업은 반드시 챙겨야 함

## 6. 배포 진행 순서 (실행 계획)

### 1단계. 무료 VPS 준비
- Oracle Cloud Always Free 계정 생성
- Ubuntu VM 생성
- 공인 IP 확보

### 2단계. 도메인 구매 및 연결
- `???.com` 도메인 구매
- DNS `A` 레코드를 서버 IP로 연결

예시:
- `@` → 서버 IP
- `www` → 서버 IP (또는 `@` CNAME)

### 3단계. 서버 기본 설정
- SSH 키 로그인 설정
- Docker 설치
- 방화벽 오픈 (`22`, `80`, `443`)

### 4단계. Kamal 배포 설정 정리
수정 대상:
- `config/deploy.yml`
  - 서버 IP
  - 이미지 이름/registry
  - 도메인/SSL 설정
- `.kamal/secrets`
  - `RAILS_MASTER_KEY`
  - 레지스트리 비밀번호(필요 시)
  - 카카오 API 키(필요 시)

### 5단계. HTTPS 활성화
- `config/deploy.yml`에서 `proxy.ssl`, `proxy.host` 설정
- `production.rb`에서 SSL 관련 옵션 활성화 필요
  - `config.assume_ssl = true`
  - `config.force_ssl = true`

### 6단계. 최초 배포
- `bin/kamal setup`
- `bin/kamal deploy`

### 7단계. 배포 후 기능 점검
- 헬스체크 `/up`
- 로그인
- 예약 생성 -> 호스트 확정(호수 지정) -> 결제
- 이미지 업로드/표시
- 재고/호수 저장

## 7. 코드 수정 최소화 전략 (실무적으로)

초기 배포에서는 아래를 **유지**하는 것을 권장:
- DB: SQLite
- 이미지 저장: Active Storage local

이유:
- 설정 변경량 최소
- 빠른 배포 가능
- 기능 검증이 쉬움

단, 이후 운영 안정화 단계에서 전환 권장:
- DB: PostgreSQL
- 파일 저장: Cloudflare R2 또는 AWS S3

## 9. 현재 프로젝트 상태에 대한 최종 판단

### 좋은 점
- Rails 구조가 명확함 (MVC)
- 도메인 기능이 많이 구현됨
- Kamal/Docker 배포 기반이 이미 있음
- 배포 전략 수립이 쉬운 편

### 주의할 점
- 현재 운영 DB가 SQLite
- 이미지가 로컬 저장
- `config/deploy.yml`는 아직 샘플값
- SSL 설정(`production.rb`)은 아직 주석 상태

### 결론
- **코드 수정 최소화 + 무료 서버 + 도메인 연결** 목표는 달성 가능
- 가장 현실적인 방법은 **무료 VPS + Kamal 배포**
- 이후 안정화 단계에서 Postgres/S3 계열로 업그레이드 권장
